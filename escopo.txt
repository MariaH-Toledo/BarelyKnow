eu preciso fazer um quiz para apresentar na semana de itinerários, faço informática para internet e estou finalizando o terceiro ano do ensino médio.
algumas informações sobre o quiz: assim que o usuários entrar na página, devem conter dois botões, um para entrar em uma sala criada e outro para você criar uma sala.
se escolher criar uma sala, vai aparecer um pop up com um input para o usuário colocar o nome que vai usar, depois disso deve aparecer um pop up para ele definir a categorias das perguntas (as perguntas serão definidas no banco de dados), quantas rodadas terá a sala que está sendo criada e definir em quantos segundos os jogadores poderão respponder cada pergunta.
se escolher o de entrar em uma sala, vai aparecer um pop up com um input para o usuário colocar o nome que vai usar, depois disso aparecerá um input para ele colocar o código da sala que será gerado assim que o host de uma sala criar um sala.
assim que o host criar a sala, existirá um tipo de lobby, onde o host terá o botão de fechar sala (cortando a conexão de todos os jogadores que entram na sala para jogar) e um botão de iniciar a partida (que só poderá ser iniciada se tiver o host e mais um jogador).
enquanto que para os jogadores convidados terá apenas o botão de saida da sala e uma mensagem dizendo que está esperando o host iniciar a partida.
a mecânica do jogo é bem simples, estilo kahoot, assim que a partida iniciar, todos os jogadores (incluindo o host) terão a visão da pergunta e quatro opções de alternativa, terá também a contagem regressiva em segundos (que será definida pelo host, que pode ser entra 10, 15 ou 20 segundos), as perguntas eu gostaria que fossem geradas por uma API do gemini, que as fará na hora.
meu professor criou no banco um mecanismo de uma tabela que guardará perguntas e alternativas, onde na segunda coluna, que seria alternativa 1, ficará sempre a resposta certa, mas na hora de aparecer para os usuários, elas seram embaralhadas, para não ter risco de aparecer sempre a resposta na alternatica A, por exemplo.
os pontos serão contados conforme a demora para responder, quem responder mais rápido e certo, ganhará mais pontos, conforme demora, recebe menos pontos. e se errar não recebe nada, estilo kahoot mesmo.
quando a partida acabar, aparecerá um rank com os 15 melhores da partida. na tela de host terá os botões de iniciar a sala novamente, onde mandará eles e os usuários convidados para o lobby de novo, ou o botão de sair e fechar sala, onde jogará ele e os outros jogadores para a tela onde escolhe se quer criar ou entrar em uma sala.
na tela de rank para versão de usuário convidado, terá só o botão de sair, caso queira sair da sala e será jogado para tela com as opções de criar sala ou entrar em uma sala.
